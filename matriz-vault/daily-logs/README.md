# Daily Logs

Diario informal de trabajo. Un archivo por día, formato `YYYY-MM-DD.md`.

Usar template: `tpl-daily-log`

## Qué anotar

- Foco del día y si se cumplió
- Aprendizajes (lo que te sorprendió o descubriste)
- Blockers y cómo los resolviste
- Contexto para retomar mañana

## Qué NO anotar

- Cada línea de código que cambiaste (eso es git)
- Status updates formales (eso es Linear)
- Documentación técnica (eso va en `knowledge/` o `domain/`)
